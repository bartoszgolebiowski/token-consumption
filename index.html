Architecture

Classic Chat vs. Agent Mode

How these tools operate determines how tokens are consumed. The structural difference directly impacts overall cost.

💬

Classic AI Chat

Linear & Predictable: One-shot request-response. The system pauses and remains idle after responding. Costs grow directly based on manual inputs and printed output.

🔄

GitHub Copilot (Agent Mode)

Autonomous ReAct Loops: Reason-Act-Observe cycle. Works behind the scenes to execute plans, review logs, and run code, continuously rebuilding context.

Highlight the difference between user-driven prompts (where the developer is in full control of the token usage) and autonomous loops (where the agent runs multiple turns automatically without user approval).
Deep Dive

The ReAct Loop Multiplier

Why does Agent Mode consume so many tokens? The answer lies in how prompts are dynamically constructed behind the scenes.

🔁 Repeated Context Processing

In each subsequent loop iteration, the model must reread the entire context: the original task, the history of previous steps, and the logs of completed operations.

📂 Multidirectional "Puffing"

The agent independently decides whether to include new data. It can read up to 5 additional source files without the user's knowledge if deemed necessary.

Explain context accumulation and "puffing". This is why a simple one-line prompt to an agent can turn into a 100k token execution if it starts reading multiple files and logs in a loop.

Classic Chat: Single Transaction

In a standard AI chat interface, you interact in a single, distinct loop. You pay only once for the context you submit and the direct completion response.

The Chat Formula

Total = Context + Prompt + Completion
Prompt + Context Files 10,000 Context + 1,000 Prompt
11,000 Tokens
Total Billed: ~11,000 Tokens
Explain that this is the baseline. When a user submits a prompt, it's a static transaction. There is no automated background work, meaning token spend stops immediately after the response is generated.
Visualizing Token Accumulation

Context Inflation: How Costs Accumulate

In classic chat, you pay for a single context exchange. In agentic runs, you pay for every loop iteration containing the sum of all previous context, steps, and tool logs.

The Accumulation Formula

Total = Sum(Context + History + Logi)
Loop 1: Initial Context 11,000 Tokens
⬇️
Loop 2: Context + Step 1 Logs 12,000 Tokens
⬇️
Loop 3: Context + History + Step 2 Logs 13,500 Tokens
⬇️
Loop 10: Full History + Accumulated Logs 20,000 Tokens
Total Billed: ~155,000 Tokens
Explain that this diagram visualizes context compounding. At Loop 10, the developer is paying for Loop 1, 2, 3... all the way to 10 combined, because LLMs are stateless and every action requires sending the full history back.
GitHub Copilot

Base Prompt Overheads

Every transaction carries an immutable system payload of **~10,000 tokens** (system rules, boundaries) that cannot be skipped. The baseline overhead then scales based on the active mode:

💬 Ask Mode

~13,000 Tokens

Base instructions + UI state + workspace indices loaded out-of-the-box.

🤖 Agent Mode

~18,000 Tokens

Ask Mode baseline + tool definitions + ReAct loop instructions (without MCPs).

Explain that the 10k token system instructions are the core foundation. Moving from Ask to Agent mode adds another 5k tokens of planner and tool schemas, creating a raw 18k token starting point before any files are read.
Extensibility

Model Context Protocol Overhead

Integrating Model Context Protocol (MCP) servers and IDE extensions adds an additional "brick" of tokens loaded into the starting context of every execution turn.

🤖 Copilot Agent Base

18,000 Tokens

Out-of-the-box system prompt baseline for planning and execution boundaries.

➕ Custom MCPs & Extensions

+0 to 10,000+ Tokens

Variable token schemas added by custom MCPs (like Jupyter) and active IDE extensions.

Total Start Cost: 18,000 to 28,000+ Tokens
Explain that the 18k base from Copilot is just the starting point. When you load complex MCP tools like Jupyter or database connectors, their schemas inject an extra 0-10k+ tokens into the system prompt, inflating the initial payload.
Optimization Alternative

On-Demand Agent Skills

Extend agent capabilities with lightweight JavaScript functions. You only load and execute them when required, bypassing the continuous schema overhead of persistent MCP servers.
